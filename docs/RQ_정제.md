# 연구 질문 및 가설 정제

## 1. 연구 질문 (Research Question)

> **RQ**: 불확실한 실내 길찾기에서 스마트글래스-Beam Pro 하이브리드 사용이 글래스 단독 사용 대비 기기 전환 행동과 신뢰 보정에 미치는 영향은 무엇인가?

### 하위 질문
- **RQ-a**: 하이브리드 조건에서 참가자가 Beam Pro 정보 허브를 참조하는 패턴(빈도, 시점, 지속시간, **콘텐츠 유형별 접근 분포**)은 어떠한가?
- **RQ-a'**: 어떤 유형의 Beam Pro 콘텐츠(POI 상세, 상황 카드, 비교 카드, 인터랙티브 맵)가 미션 수행 정확도와 가장 강한 연관을 보이는가? *(v2.1 신규)*
- **RQ-b**: 기기 조건에 따라 불확실성 지점에서의 신뢰 보정(trust calibration)은 어떻게 달라지는가?
- **RQ-c**: 기기 조건에 따라 길찾기 수행 효율(시간, 정지)은 어떻게 달라지는가?

---

## 2. 가설 (Hypotheses)

### H1: 신뢰 보정 (Trust Calibration) — v2 수정

> 하이브리드 조건에서 불확실성 트리거 발생 시 확신도-정확도 상관(calibration)이 Glass Only 조건보다 높다. 즉, Hybrid 사용자는 자신의 확신도가 실제 미션 수행 결과와 더 잘 일치한다.

**근거**:
- Lee & Moray (2004) — 자동화 시스템의 오류 발생 시, 대안 정보 소스에 접근 가능하면 신뢰 회복이 빠르다. 하이브리드 조건에서 Beam Pro는 글래스 내비게이션의 불확실성을 보완하는 대안 정보 소스 역할을 한다.
- de Vries et al. (2003) — 사용자가 시스템의 추론 과정을 추가 디스플레이로 확인할 수 있을 때, 시스템 오류에 대한 신뢰 저하가 완화된다.

**측정**:
- 1차: **확신도-미션 정확도 상관계수 (calibration index)** — 참가자별, 조건별 피어슨 r
- 2차: 경유지별 확신도(1-7) 변화량 — 트리거 전후 차이 (4종 트리거 적용)
- 3차: 사후 시스템 신뢰 척도 (Jian et al., 2000 기반)

**예상 패턴**:
```
확신도
7 ─┐
   │  ──Hybrid──
6 ─┤
   │       ╲
5 ─┤        ── (완만한 감소 후 빠른 회복)
   │
4 ─┤  ──Glass──     (급격한 감소 후 회복 지연)
   │
   └──────────────────────────→ 웨이포인트
      WP1  WP2  *트리거*  WP4  WP5  *트리거*  WP7  WP8
```

**통계**: Paired t-test (또는 Wilcoxon signed-rank) — 2조건 비교

### H2: 수행 효율 (Task Performance) — v2 확장

> Hybrid 조건은 Glass Only 대비 미션 정확도가 높다. 단, 과제 완료 시간은 Beam Pro 참조로 인해 동등하거나 약간 길 수 있다.

**근거**:
- Rehman & Cao (2017) — HMD 기반 내비게이션은 hands-free 특성으로 이동 중 정보 접근이 가능하여 이동 효율이 증가한다.
- Müller et al. (2021, BISHARE) — 핸드헬드 디바이스-HMD 간 상호 전환이 가능할 때, 사용자가 상황에 따라 최적 기기를 선택하여 과제 수행 효율이 개선된다.

**측정**:
- 1차: **미션 정확도** — 전체 및 미션 타입별 (A/B/C) 정답률
- 2차: 총 과제 완료 시간 (초)
- 3차: 정지 횟수 및 총 정지 시간 (초)
- 4차: 미션별 소요시간 (MISSION_START ~ VERIFICATION_ANSWERED)

### H3: 교차검증 행동 (Cross-Verification) — v2 신규

> 하이브리드 조건에서 불확실성 트리거 발생 시 Beam Pro 참조 확률이 비트리거 구간 대비 유의하게 높으며, 트리거 시점의 Beam Pro 참조는 미션 정확도 향상과 정적 상관을 보인다.

**근거**:
- 분산 인지 이론(Hutchins, 1995) — 불확실성이 높아질 때 사용자는 추가 인지 자원(외부 도구)을 능동적으로 동원한다.
- Complementary computing — 하이브리드 시스템에서 사용자가 두 기기의 상보적 강점을 상황에 맞게 활용하는 전략.

**측정**:
- 1차: **교차검증 지수(CVI)** — 트리거 구간 Beam Pro 참조율 ÷ 전체 참조율 (>1이면 트리거 시 더 많이 참조)
- 2차: Beam Pro 참조 후 미션 정확도 vs 미참조 시 미션 정확도
- 3차: **콘텐츠별 CVI** — POI 상세/상황 카드/비교 카드/맵 각각의 트리거 구간 접근률 ÷ 전체 접근률 *(v2.1 신규)*
- 4차: **정보 활용도** — 콘텐츠 유형별 열람 후 미션 정확도 상관 *(v2.1 신규)*

### 탐색적 분석 (Exploratory) — v2 확장

기기 전환 패턴 분석 — **가설 없이 탐색적으로 기술**:
- 하이브리드 조건에서 참가자가 **언제** Beam Pro를 참조하는가 (불확실성 트리거 전후? 특정 웨이포인트?)
- **어디서** 전환이 발생하는가 (교차로? 특정 복도 구간?)
- **얼마나 오래** Beam Pro를 참조하는가 (평균 전환 지속시간)
- 경로 진행에 따른 전환 빈도 변화 (학습 효과 존재 여부)

**v2 추가 탐색 항목**:
- **미션 타입별 기기 전환 패턴**: 미션 A/B/C 각각에서 Beam Pro 참조 빈도·시간 비교
- **트리거 유형별 반응 차이**: T1(안내 열화)/T2(정보 불일치)/T3(해상도 부족)/T4(안내 부재) 각각의 확신도 변화·반응시간
- **검증 행동 분류**: Beam Pro 참조가 사전적(proactive) vs 반응적(reactive)인지 분류
- **학습 효과**: 1차→2차 경로에서 기기 전환 전략 변화
- **확신도 보정(calibration) 곡선**: 확신도 수준별 실제 정확도 대응

**v2.1 추가 탐색 항목 (Beam Pro 정보 허브)**:
- **콘텐츠 접근 패턴 분류**: 참가자의 Beam Pro 콘텐츠 접근 전략 유형화 (맵 중심 / 카드 중심 / 혼합형)
- **카드 자동 표시 반응**: 자동 표시된 정보 카드의 열람율, 무시율, 열람 시간
- **미션별 콘텐츠 선호**: 미션 타입 × 콘텐츠 유형 히트맵 (어떤 미션에서 어떤 콘텐츠를 더 참조하는가)
- **정보 과부하 징후**: Beam Pro 정보 접근량과 NASA-TLX 점수 간 상관, 과도한 정보 접근이 오히려 수행을 저하시키는 역U자형 관계 여부

## 3. 이론적 프레임워크

### 3.1 분산 인지 (Distributed Cognition)

Hutchins (1995)의 분산 인지 이론에 따르면, 인지 과정은 개인 내부에만 국한되지 않고 환경의 도구와 표상에 걸쳐 분산된다.

**본 연구에의 적용**:
- **Glass Only 조건**: 인지 자원이 하나의 기기-사용자 인터페이스에 집중
  - hands-free + head-up, but 제한된 정보 표현
- **하이브리드 조건**: 인지 과정이 두 기기에 걸쳐 분산
  - 글래스 = WHERE: 실시간 방향 안내 (ambient, peripheral)
  - Beam Pro = WHAT: 정보 허브 (focal, on-demand)
    - POI 상세: 목적지/주변 시설의 상세 정보 (수용인원, 장비 등)
    - 상황 카드: 현재 위치/미션 맥락에 맞는 정보 자동 제공
    - 비교 카드: 미션 C의 비교 판단용 구조화된 참고 자료
    - 인터랙티브 맵: 건물 구조 + POI 마커 기반 공간 탐색
  - 사용자가 상황에 따라 정보 소스를 전환 → **적응적 인지 분산**

### 3.2 신뢰 보정 (Trust Calibration)

Lee & See (2004)의 신뢰 프레임워크에 따르면, 자동화 시스템에 대한 신뢰는 시스템의 수행과 사용자의 기대 간 일치도에 의해 보정된다.

**본 연구에의 적용**:
- **신뢰 위반 (Trust Violation)**: 불확실성 트리거(모호한 교차로, 오프셋 화살표)가 시스템 신뢰를 저하시키는 이벤트
- **보정 기제**: 하이브리드 조건에서 대안 기기를 통한 교차 검증이 신뢰 보정을 촉진
- **과보정 vs 적정 보정**: 글래스 단독에서는 대안 없이 신뢰가 과도하게 저하될 수 있음

### 3.3 두 프레임워크의 통합

```
┌──────────────────────────────────────────────────────────────┐
│              분산 인지 (Distributed Cognition)                  │
│                                                              │
│  글래스 (Ambient — WHERE)  ←──전환──→  Beam Pro (Focal — WHAT) │
│   · 방향 화살표                    · 인터랙티브 맵 + POI 마커   │
│   · 실시간 안내                    · POI 상세 카드              │
│                                    · 상황 정보 카드             │
│                                    · 비교 카드 (미션 C)         │
│                                    · 미션 참조 패널             │
│                                                              │
│          ↓ 불확실성 트리거 발생 시 ↓                             │
│                                                              │
│              신뢰 보정 (Trust Calibration)                     │
│                                                              │
│  단일 기기: 보정 정보 부재 → 신뢰 급락 또는 과신                  │
│  하이브리드: 다양한 콘텐츠로 교차 검증 → 적응적 신뢰 보정          │
└──────────────────────────────────────────────────────────────┘
```

## 4. 종속변수 조작적 정의

| 변수 | 조작적 정의 | 측정 방법 | 단위 |
|------|------------|----------|------|
| **경유지별 확신도** | 각 웨이포인트 도달 시 "현재 올바른 경로에 있다고 얼마나 확신하는가?"에 대한 자기보고 | 7점 리커트 척도 (1=전혀 확신 없음, 7=매우 확신) | 점 (1-7) |
| **확신도 변화량** | 불확실성 트리거 직전 웨이포인트 확신도 - 직후 웨이포인트 확신도 | 경유지별 확신도 차이 계산 | 점 차이 |
| **시스템 신뢰** | 각 조건 사용 후 내비게이션 시스템에 대한 전반적 신뢰도 | Jian et al. (2000) Trust Scale 기반 7문항, 7점 리커트 | 평균 점수 (1-7) |
| **과제 완료 시간** | 경로 시작 신호부터 마지막 웨이포인트 도달까지의 경과 시간 | 이벤트 로그 ROUTE_START ~ ROUTE_END 타임스탬프 차이 | 초 (s) |
| **정지 횟수** | 경로 수행 중 2초 이상 이동이 멈춘 횟수 | 이벤트 로그 PAUSE_START 이벤트 카운트 | 회 |
| **총 정지 시간** | 경로 수행 중 정지 상태의 누적 시간 | PAUSE_START ~ PAUSE_END 구간 합산 | 초 (s) |
| **기기 전환 빈도** | 하이브리드 조건에서 Beam Pro 화면을 켠 횟수 | BEAM_SCREEN_ON 이벤트 카운트 | 회 |
| **평균 전환 시간** | Beam Pro 화면 ON부터 OFF까지의 평균 지속 시간 | BEAM_SCREEN_ON ~ BEAM_SCREEN_OFF 구간 평균 | 초 (s) |
| **NASA-TLX** | 각 조건 수행 직후 인지 부하 자기보고 | NASA-TLX 한국어판 6개 하위척도, 각 21점 | 가중 평균 점수 |
| **미션 정확도** | 각 미션의 검증 질문 정답 여부 (v2 신규) | 검증 UI에서 정답/오답 자동 기록 | 정답률 (0-1) |
| **미션별 소요시간** | 미션 브리핑~검증 완료 경과 시간 (v2 신규) | MISSION_START ~ VERIFICATION_ANSWERED 타임스탬프 차이 | 초 (s) |
| **확신도 보정 지수** | 확신도와 미션 정확도 간 상관 (v2 신규) | 참가자별 피어슨 상관계수 | r (-1~1) |
| **교차검증 지수 (CVI)** | 트리거 구간 Beam Pro 참조율 ÷ 전체 참조율 (v2 신규) | 이벤트 로그에서 산출 | 비율 |
| **트리거 반응시간** | 트리거 활성화~첫 반응 행동까지 시간 (v2 신규) | TRIGGER_ACTIVATED ~ 다음 행동 이벤트 | 초 (s) |
| **난이도 평정** | 각 미션 완료 후 주관적 난이도 (v2 신규) | 7점 리커트 척도 | 점 (1-7) |
