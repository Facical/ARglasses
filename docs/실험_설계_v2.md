# 실험 설계 v2: 미션 기반 길찾기

> **버전**: v2.1 — 2조건(Glass Only vs Hybrid) + 단일 층 설계. 미션 기반 태스크로 재설계.

---

## 1. 문제 진단: 기존 설계의 한계

### 1.1 핵심 문제

기존 설계(v1)는 3조건(Beam Pro Only / Glass Only / Hybrid) 비교였으나, 근본적 문제가 있었다:

| 문제 | 설명 |
|------|------|
| **Beam Pro Only와 Glass Only는 비교 불가** | 핸드헬드(head-down) vs HMD(head-up)는 폼팩터 자체가 달라, 차이가 기기 전략이 아닌 입력 모달리티 차이를 측정하게 됨 |
| **수동적 태스크** | 화살표만 따라가면 되므로 참가자의 능동적 판단이 불필요 |
| **Beam Pro 참조 동기 부재** | Hybrid 조건에서 Beam Pro를 꺼낼 구조적 이유가 없음 |
| **다층 구조의 구현 복잡도** | AR 트래킹이 층간 전환에서 불안정, 층별 평면도 전환 구현 복잡 |

### 1.2 해결 방향

1. **2조건 비교**: Glass Only vs Hybrid — 두 조건 모두 글래스가 기본 인터페이스이므로 기저 UX가 동일. 차이는 오직 "Beam Pro 보조 접근 가능 여부"
2. **미션 기반 태스크**: 단순 경로 추종 → 정보 탐색 + 확인이 필요한 미션
3. **단일 층**: 한 개 층의 복잡한 복도 구조 활용, 구현 복잡도 대폭 감소

## 2. 실험 설계 개요

| 항목 | 내용 |
|------|------|
| **설계** | 2 (기기 조건) × Within-subjects |
| **기기 조건** | Glass Only (G) / Hybrid (H) |
| **참가자 수** | 24명 (2순서 × 12명) |
| **균형화** | 단순 역균형 (G→H, H→G) |
| **경로 수** | 2개 (조건당 1개) |
| **경로 환경** | 단일 층 (복도, 교차로 포함) |
| **소요 시간** | 참가자당 약 55분 |

### 2.1 조건 정의

#### Glass Only 조건
- 글래스에 AR 방향 화살표 표시
- Beam Pro 화면 잠금 (접근 불가)
- 참가자는 화살표 + 환경 관찰만으로 미션 수행

#### Hybrid 조건
- 글래스에 AR 방향 화살표 표시 (Glass Only와 동일)
- Beam Pro를 들어올리면 **정보 허브(Information Companion)** 활성화 (lift-to-wake)
  - 3-레이어 정보 구조: ① 인터랙티브 맵 ② 상황 정보 카드 ③ 미션 참조 패널
  - 글래스(WHERE to go) + Beam Pro(WHAT to know) 역할 분리
- 사용 시점과 빈도는 참가자 자유

### 2.2 핵심 비교 논리

두 조건의 **유일한 차이는 Beam Pro 정보 허브 접근 가능 여부**이다:

```
기존 (v2.0):
Glass Only:   글래스(AR 화살표) ─────────────────────→ 미션 수행
Hybrid:       글래스(AR 화살표) + Beam Pro(지도)  ────→ 미션 수행   ← 정보 중복

재설계 (v2.1):
Glass Only:   글래스(WHERE: 방향 화살표) ─────────────→ 미션 수행
Hybrid:       글래스(WHERE: 방향 화살표)
              + Beam Pro(WHAT: 정보 허브) ────────────→ 미션 수행   ← 정보 상보성
                         ↑
                  이것의 효과를 측정
```

**설계 원칙**: 모든 미션은 Glass Only에서도 수행 **가능**해야 함 (현장 관찰로). Beam Pro 정보 허브는 "더 쉽고 정확"하게 만들 뿐, 없으면 불가능한 것은 아님.

## 3. 태스크 재설계: 미션 기반 길찾기

### 3.1 설계 철학

참가자가 단순히 화살표를 따라가는 것이 아니라, **목적지에서 특정 정보를 확인·판단해야 하는 미션**을 수행한다. 이를 통해:

1. 참가자가 **능동적으로 정보를 탐색**해야 하므로, Beam Pro 보조 접근의 가치가 체감됨
2. 불확실한 상황에서 **Beam Pro를 참조할 동기**가 자연스럽게 발생
3. 미션 정확도라는 **객관적 수행 지표**가 추가됨

### 3.2 미션 타입 정의

#### 미션 타입 A: 방향 안내 + 검증 (Direction & Verify)

- **구조**: 내비게이션이 목적지까지 안내 → 도착 후 확인 질문에 답해야 완료
- **예시**:
  - "회의실 302를 찾아가세요. 도착 후 회의실의 최대 수용 인원을 보고하세요."
  - "자판기 코너로 이동하세요. 자판기 옆에 있는 비상구 번호를 확인하세요."
- **요구 인지**: 내비게이션 정보 + 환경 관찰
- **Beam Pro 제공 정보 (Hybrid)**: POI Detail Card — 목적지 회의실의 수용인원, 장비 목록, 호실번호 등 상세 정보
- **경로당 2개**

#### 미션 타입 B: 모호한 의사결정 (Ambiguous Decision)

- **구조**: 내비게이션 안내가 불확실한 지점에서 스스로 방향 판단
- **예시**:
  - 교차로에서 화살표가 두 방향의 중간을 가리킴 → 어느 쪽으로 갈지 판단
  - 안내가 "이 구역 어딘가"만 표시 → 세부 위치는 환경 단서로 탐색
- **요구 인지**: 불확실성 하의 의사결정, 다중 정보원 참조
- **Beam Pro 제공 정보 (Hybrid)**: 교차로 주변 POI 마커 + Contextual Info Card — 각 방향에 어떤 시설이 있는지 상황 정보 제공
- **경로당 2개**

#### 미션 타입 C: 정보 통합 (Information Integration)

- **구조**: 기기 정보와 환경 관찰을 결합해야 답을 도출
- **예시**:
  - "지도에 표시된 '세미나실 A'로 이동하세요. 도착 후, 세미나실 A와 B 중 프로젝터가 있는 곳을 보고하세요."
  - "복도 양쪽의 회의실 중 창문이 있는 회의실 번호를 찾으세요."
- **요구 인지**: 기기 정보와 환경 정보의 교차 참조
- **Beam Pro 제공 정보 (Hybrid)**: Comparison Card — 비교 대상의 기본 속성(장비, 크기, 용도) 표 제공 + 현장 핵심 확인 포인트 안내
- **경로당 1개**

### 3.3 미션 배치 원칙

```
경로 시작점
    │
    ├── WP01: 미션 A1 브리핑 → 이동 → WP02: 미션 A1 검증
    │
    ├── WP03: [트리거] + 미션 B1 (모호한 갈림길)
    │
    ├── WP04: 미션 A2 브리핑 → 이동 → WP05: 미션 A2 검증
    │
    ├── WP06: [트리거] + 미션 B2 (불확실한 구간)
    │
    ├── WP07: 미션 C1 (정보 통합) → WP08: 미션 C1 검증
    │
경로 종점
```

- 경로당 **8개 웨이포인트**, **5개 미션** (A×2, B×2, C×1), **2개 트리거**
- 미션 B는 트리거와 결합하여 불확실성 상황에서의 의사결정을 유발

## 3-A. Beam Pro 정보 허브 설계 (Information Companion)

### 3-A.1 설계 배경

기존 설계에서 Beam Pro는 2D 지도만 표시했으나, 글래스 화살표가 이미 방향 안내를 하므로 **지도는 중복 정보**에 불과했다. 이를 해결하기 위해 Beam Pro를 **정보 허브(Information Companion)**로 재설계하여, 글래스(WHERE to go)와 Beam Pro(WHAT to know)의 역할을 분리한다.

- **BISHARE(2020)**: "2D content on phone, 3D/spatial on HMD" 원칙 적용
- **AReading(2025)**: 폰의 고해상도(440-570 PPI)가 텍스트/상세 콘텐츠에 압도적 우위 → 상세 정보 표시에 활용

### 3-A.2 3-레이어 정보 구조

#### Layer 1: Enhanced Map (인터랙티브 맵)
- 건물 평면도 + 경로 오버레이 (유지)
- **POI 마커 추가** (회의실, 자판기, 화장실, 비상구 등)
- 마커 탭 → POI 상세 정보 패널 표시
- 현재 웨이포인트 기반 자동 줌
- 확대/축소 제스처 지원

#### Layer 2: Contextual Information Cards (상황 정보 카드)
- 현재 위치/미션에 따라 **자동 표시**되는 정보 카드
- 카드 유형:
  - **POI Detail Card**: 방 상세 (수용인원, 장비, 호실번호)
  - **Sign/Poster Card**: 멀리 있는 안내판/포스터의 고해상도 텍스트
  - **Comparison Card**: 미션 C의 비교 판단용 참고 자료
  - **Landmark Card**: 주변 랜드마크 설명
- 카드는 자동 표시(push) 또는 사용자 탭(pull) 모두 가능

#### Layer 3: Mission Reference Panel (미션 참조 패널)
- 미션 브리핑 재열람
- 미션 타입별 보조 정보 (A: 목적지 POI 힌트, B: 주변 지도 정보, C: 비교 속성 목록)

### 3-A.3 미션 타입별 Beam Pro 정보 허브 가치

| 미션 | Glass Only 경험 | Hybrid에서 Beam Pro 추가 가치 |
|------|----------------|------------------------------|
| **A** (방향+검증) | 도착 후 문 앞 안내판을 직접 읽어서 검증 | POI 상세로 수용인원/장비 등 즉시 확인 → 더 빠르고 정확 |
| **B** (모호한 의사결정) | 환경 단서만으로 방향 추측 | 교차로 주변 POI 정보 + 상황 카드로 **정보 기반 판단** |
| **C** (정보 통합) | 양쪽 직접 방문해서 비교 | 비교 카드로 기본 정보 확인 + 현장에서 핵심 속성만 확인 |

### 3-A.4 트리거 유형별 Beam Pro 정보 허브 가치

| 트리거 | Beam Pro 정보 허브 기대 행동 |
|--------|---------------------------|
| **T1** (안내 열화) | 인터랙티브 맵 줌 → 현재 위치 및 경로 확인 |
| **T2** (정보 불일치) | POI 상세 + 상황 카드 → 화살표/표지판 불일치 해소 |
| **T3** (해상도 부족) | 인터랙티브 맵 줌 → 정확한 방향 확인 |
| **T4** (안내 부재) | POI 상세 → 목적지 정확한 위치 파악 |

## 4. 불확실성 트리거 설계

### 4.1 트리거 4종

#### T1: 안내 열화 (Tracking Degradation)
- AR 화살표가 5-8초간 떨림/깜빡임 후 사라짐 (tracking loss 시뮬레이션)
- 10초 후 자동 복구
- **의도**: 시스템 신뢰 위반, 대안 기기 참조 동기 생성

#### T2: 정보 불일치 (Information Conflict)
- AR 화살표 방향과 환경 표지판 방향이 상충
- 실제로는 화살표가 정확 (표지판은 다른 목적지를 가리킴)
- **의도**: 다중 정보원 간 갈등, 교차검증 행동 유발

#### T3: 해상도 부족 (Low Resolution Guidance)
- AR 화살표가 부채꼴(±30°)로 퍼져 모호한 방향만 표시
- **의도**: 글래스 정보의 한계 체감, 상세 정보(Beam Pro 지도) 필요성

#### T4: 안내 부재 (Guidance Absence)
- 마지막 ~20m 구간에서 "목적지 근처입니다"만 표시, 화살표 제거
- Beam Pro 지도에는 정확한 목적지 핀 표시 유지
- **의도**: 자력 탐색 필요, Beam Pro 지도 가치 극대화

### 4.2 트리거 배치

| | 경로 A | 경로 B |
|---|--------|--------|
| **WP03 부근** | T1 (안내 열화) | T2 (정보 불일치) |
| **WP06 부근** | T4 (안내 부재) | T3 (해상도 부족) |

- 각 참가자는 2개 경로에서 트리거 4회 경험 (4종 각 1회)
- 역균형 배정으로 조건 × 트리거 유형 조합이 균형

## 5. 경로 설계

### 5.1 경로 사양

| 항목 | 기준 |
|------|------|
| 경로 수 | 2개 (경로 A, 경로 B) |
| 웨이포인트 수 | 8개 / 경로 |
| 미션 수 | 5개 / 경로 (A×2, B×2, C×1) |
| 트리거 수 | 2개 / 경로 |
| 총 거리 | ~200m / 경로 |
| 층 | **단일 층** |
| 총 턴 수 | 5-6회 |
| 예상 소요 시간 | ~15분 / 경로 |

### 5.2 단일 층 경로 설계 기준

한 개 층에서 충분한 복잡도를 확보하기 위한 요건:

- **T자/십자 교차로** 최소 3개 포함 (미션 B 트리거 배치용)
- **시각적으로 유사한 복도 구간** 존재 (모호한 의사결정 유발)
- **다양한 랜드마크** 배치 (회의실, 자판기, 안내판 등 — 미션 검증용)
- **경로 A와 B는 동일 층의 서로 다른 구역** 사용 (중복 방지)

### 5.3 경로 A 상세 (서쪽-북쪽 루프)

> **실험 장소**: KIT 디지털관 지하1층

**경로**: 중앙 계단 → 남쪽복도 서진 → SW교차로(좌회전) → 서쪽복도 북상 → NW교차로(우회전) → 북쪽복도 동진 → 종료

```
          NW교차로 ─── 북쪽복도 (B132, B133, B134) ─── NE교차로
            │                                              │
  서쪽복도  │  B131, B130, B129, B128          B101~B111  │  동쪽복도
  (북상)    │                                     (남하)   │
            │                                              │
          SW교차로 ─── 남쪽복도 (B127, 계단, B123~B121) ── SE교차로
                              ↑ 시작점
```

#### 웨이포인트 테이블

| WP | 위치 | 이벤트 | 미션/트리거 |
|----|------|--------|------------|
| WP01 | B123 강의실 앞 (남쪽복도) | Mission A1 브리핑 | "대강의실을 찾아가세요" |
| WP02 | B125 대강의실 앞 (서쪽) | Mission A1 검증 | 호실 번호/수용인원 확인 |
| WP03 | SW교차로 (B127 근처) | **T1** + Mission B1 | 트래킹 열화 + 시청각실 찾기 |
| WP04 | B129 대학원강의실 앞 (서쪽복도) | Mission A2 브리핑 | "대학원 강의실을 찾아가세요" |
| WP05 | B130 대학원강의실 앞 (서쪽복도) | Mission A2 검증 | 장비/호실 확인 |
| WP06 | NW교차로 (B131/B132 전환점) | **T4** + Mission B2 | 안내 부재 + 세미나실 찾기 |
| WP07 | B132 강의실 앞 (북쪽복도) | Mission C1 시작 | B132 vs B133 비교 |
| WP08 | B133 세미나실 앞 (북쪽복도) | Mission C1 검증 | 비교 결과 응답 |

## 8. 가설

### H1: 신뢰 보정 (Trust Calibration)

> Hybrid 조건에서 확신도-정확도 상관(calibration)이 Glass Only 조건보다 높다.

**측정**:
- 1차: 확신도-미션 정확도 상관계수 (calibration index)
- 2차: 트리거 전후 확신도 변화량
- 3차: 사후 시스템 신뢰 척도

**통계**: Paired t-test (또는 Wilcoxon signed-rank)

### H2: 미션 수행 (Task Performance)

> Hybrid 조건은 Glass Only 대비 미션 정확도가 높다. 단, 과제 완료 시간은 Beam Pro 참조로 인해 동등하거나 약간 길 수 있다.

**측정**:
- 1차: 미션 정확도 (전체, 타입별)
- 2차: 과제 완료 시간
- 3차: 정지 횟수 및 총 정지 시간

**통계**: Paired t-test (또는 Wilcoxon signed-rank)

### H3: 교차검증 행동 (Cross-Verification)

> Hybrid 조건에서 트리거 발생 시 Beam Pro 참조 확률이 비트리거 구간 대비 유의하게 높으며, Beam Pro 참조는 미션 정확도 향상과 정적 상관을 보인다.

**측정**:
- 1차: 교차검증 지수(CVI) — 트리거 구간 참조율 ÷ 전체 참조율
- 2차: Beam Pro 참조 후 정확도 vs 미참조 시 정확도

**통계**: 기술통계 + 카이제곱 검정

### 탐색적 분석

- **미션 타입별 기기 전환 패턴**: 어떤 미션에서 Beam Pro를 더 많이 참조하는가
- **트리거 유형별 반응 차이**: T1-T4 각각에서 확신도 변화·반응시간 비교
- **검증 행동 분류**: Beam Pro 참조가 사전적(proactive) vs 반응적(reactive)인지
- **학습 효과**: 1차→2차 경로에서 기기 전환 전략 변화
